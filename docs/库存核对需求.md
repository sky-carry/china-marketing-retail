# 库存核对平台 · 需求说明

> 更新时间：2026-07-06
> 数据库：`postgresql://postgres:postgres@localhost:5432/inventory_check`

## 1. 目标

以飞书《即时零售门店上翻明细 · 专卖店》为门店主档，核对每个专卖店、每个商品在三方的库存数量是否一致：

- **伯俊**（线下实际库存，基准值）
- **京东秒送**（线上上翻库存）
- **美团闪购**（线上上翻库存）

产出差异明细 + 门店级汇总，并把无法自动匹配的门店/商品单独列出供人工处理。

## 2. 数据源与表

| 表 | 来源 | 行数 | 角色 |
| --- | --- | --- | --- |
| `feishu_store_mapping` | 飞书·专卖店（三平台门店映射） | 273 家有效（过滤空行） | 门店主档 |
| `bojun_offline_inventory` | 伯俊线下库存.xlsx | 14,463 | 库存基准 |
| `jd_store_inventory` | 京东门店库存-新.xlsx（2026-07-08 商家商品编号补全版） | 219,830 | 京东上翻库存 |
| `meituan_store_inventory` | 美团门店库存.xlsx | 12,165 | 美团上翻库存 |
| `jd_store` / `meituan_store` | 门店档案 | 1,451 / 304 | 辅助（营业状态等） |
| `feishu_jd_outlet` / `feishu_meituan_outlet` / `feishu_region_contact` | 飞书附表 | 1,100 / 102 / 21 | 辅助（京东网点、美团网点、对接人） |

## 3. 关联规则

### 3.1 门店维度（统一维度视图 `v_dim_store`）

门店口径只此一处：`v_dim_store` = 飞书主档（过滤空行）+ 人工别名修正 + 启用状态。

- **逻辑删除**：备注为 `闭店`（22 家）/ `机场店`（7 家）的门店 `is_active=false`，不参与核对、不进看板；`京东已切自配送` 仍参与。在营门店 244 家。
- **别名修正**：`store_alias` 表（`飞书门店名 → 伯俊店仓名`，数据刷新不清空）。名字对不上的在营门店往里插一行对照即可自动接入核对。

| 方向 | 关联键 | 实测匹配情况（在营口径） |
| --- | --- | --- |
| → 伯俊 | `bojun_warehouse = bojun_offline_inventory.store_warehouse`（默认=门店名，可被别名覆盖） | 236 / 244 匹配 |
| → 京东库存 | `jd_id = jd_store_inventory.store_code` | 全部匹配 |
| → 美团库存 | `meituan_id = meituan_store_inventory.store_id` | 有库存数据即匹配 |

对不上伯俊店仓名的在营门店 8 家 → 「门店未匹配清单」（排除闭店/机场店后从 32 家降到 8 家——原先对不上的大多是闭店门店）。

### 3.2 商品维度（主键：伯俊货号 `product_code`）

| 平台 | 关联键 | 实测匹配情况 |
| --- | --- | --- |
| 京东 | `merchant_product_code`（商家商品编号） | 131 个编码中 111 个匹配（2026-07-08 换用补全版后从 33/122 提升） |
| 美团 | `internal_sku_code`（店内码/货号） | 105 个编码中 98 个匹配 |

匹配不上的平台商品 → 输出到「商品未匹配清单」（附条码），原因主要有：

- 京东侧存在 `320481804` 这类京东自有编码，只带条码，而**伯俊表无条码列**，需补商品主数据（货号↔条码）后才能自动匹配；
- 部分编码是伯俊货号格式但当前伯俊快照中不存在（如 `1105050133`）；
- 录入错误（如美团 `105050081` 疑似丢首位）、赠品行（"随单赠送礼品袋"）。

## 4. 对比口径（已确认）

| 项 | 取值 | 说明 |
| --- | --- | --- |
| 伯俊库存 | `stock_qty`（库存数量） | 存在负库存（如 -22），单独标记 |
| 京东库存 | `available_stock`（可用库存） | 上翻同步的是可售数，不用现货 `onhand_stock` |
| 美团库存 | `stock_qty`（库存） | — |
| 粒度 | 门店 × 货号 | 已验证平台侧同店同品无重复行；伯俊侧用 SUM 聚合兜底 |
| 商品状态 | 不过滤，`jd_product_status` 作为参考列输出 | 下架商品也参与核对，便于发现"下架但线下有货"的情况 |
| 门店范围 | 仅 `v_dim_store.is_active` 在营门店 | 备注闭店/机场店的 29 家逻辑删除 |
| 虚拟库存 | 平台库存 ≥ 99 标记为 `平台虚拟库存`，不算差异 | 平台侧大量 99/999/9999/99999 占位值（门店防售罄设的假库存），阈值可在视图中调整 |

## 5. 输出定义

### 5.1 核对明细（视图 `v_recon_detail`）

每行 = 门店 × 货号：

| 列 | 说明 |
| --- | --- |
| `customer_name` | 客户（公司，飞书主档"客户名称"，空值归为"（未填客户）"） |
| `store_name` | 门店（飞书主档口径） |
| `product_code` / `product_name` | 货号 / 品名（优先伯俊品名） |
| `bojun_qty` | 伯俊库存 |
| `jd_qty` / `mt_qty` | 京东 / 美团库存；**NULL = 该平台未上架**（与 0 有货区分） |
| `jd_diff` / `mt_diff` | 平台库存 − 伯俊库存；正数 = 平台多翻（超卖风险），负数 = 平台少翻（漏上） |
| 维护率（看板/Excel 计算列） | 明细行级：平台库存 ≥ 伯俊 → 100%，否则 平台 ÷ 伯俊；未上架或伯俊 ≤0 无基数 → 空。客户/门店汇总级：Σmin(平台,伯俊) ÷ Σ伯俊（超翻不掩盖缺翻）；汇总差异 = Σ(平台−伯俊)，只累计伯俊有记录且非虚拟库存的行。看板明细不展示状态列（状态用于筛选，品名悬浮可见） |
| `jd_product_status` | 京东商品状态（上架/下架，参考） |
| `flag` | 状态标记：`三方一致` / `存在差异` / `伯俊负库存` / `仅平台有货(伯俊无此品)` / `平台均未上架` / `平台虚拟库存` |

### 5.2 门店汇总（视图 `v_recon_store_summary`）

按门店统计：品数、京东/美团上架数、一致数、差异数——作为平台首页概览，点开看明细。

### 5.2b 客户汇总（视图 `v_recon_customer_summary`）

按客户（公司）统计：门店数、明细行、可核对行（至少一个平台已上翻）、三方一致、存在差异、分平台上翻/相等数。看板据此计算：

- **合格率** = 三方一致 ÷ 可核对行（衡量该客户线上库存翻得准不准）
- **京东/美团一致率** = 该平台已上翻行中库存与伯俊相等的比例

### 5.2c 网点库存保障（视图 `v_outlet_guard` / `v_outlet_guard_summary`，看板「网点保障」Tab）

规则（A 口径，2026-07-16 确认）：某公司所有**在营专卖店**的伯俊库存总和（按货号，仅总和 > 0 的货号），应 ≤ 该公司**每一个**启用状态京东网点在京东的可用库存；网点未上架该货号视为不足。

- 关联：`feishu_store_mapping.customer_name` = `feishu_jd_outlet.dealer`（精确匹配；对不上的经销商如"鼎信"、脏数据"379102"、无线下店的中免——排除，已确认）
- 网点范围：`feishu_jd_outlet.store_status = '启用'`
- 看板：左表 公司×货号（线下总和/网点数/达标网点/最小库存/最大缺口，默认只看不达标），点击行右侧按需加载每个网点的库存明细；「下载 Excel」含"网点保障汇总"sheet

### 5.3 未匹配清单

- `v_store_unmatched`：在营门店按名字（含别名修正后）对不上伯俊店仓的（8 家）
- `v_product_unmatched`：平台侧编码对不上伯俊货号的（含条码、涉及门店数、库存合计）

### 5.4 视图落地

四个视图的定义在 [`sql/02_核对视图.sql`](../sql/02_核对视图.sql)，八张表的建表语句在 [`sql/01_建表.sql`](../sql/01_建表.sql)，均已在库中创建；重跑入库脚本（[`etl/load_excel.py`](../etl/load_excel.py)）刷新数据后视图结果自动更新。

### 5.5 核对结果快照（2026-07-08 数据，京东补全版，在营门店口径）

明细共 28,543 行（门店×货号），覆盖 243 家在营门店（已排除闭店/机场店 29 家）：

| 状态 | 行数 | 说明 |
| --- | --- | --- |
| 仅平台有货(伯俊无此品) | 17,116 | 平台有上架但当前伯俊快照该店无此货号记录 |
| 存在差异 | 7,653 | 真实数量不一致，核对重点 |
| 平台均未上架 | 2,748 | 伯俊有货但京东、美团都没挂 |
| 平台虚拟库存 | 541 | 平台挂 99/999/9999 占位值 |
| 三方一致 | 430 | — |
| 伯俊负库存 | 55 | 线下账面异常 |

对比 07-06 旧京东表：京东可识别上翻行从约 7,000 涨到 25,530，"平台均未上架"从 5,177 降到 2,748（原来是编号缺失导致误判未上架）；三方一致减少是因为京东侧新识别的行大多与伯俊不相等，转入了"存在差异/仅平台有货"。

## 7. 附：数据质量备忘

- `feishu_store_mapping` 表 1,687 行中约 1,400 行是飞书表格的空行，查询时必须过滤 `store_name IS NOT NULL`
- `jd_enable_status` / `verification_status` 两列源表为 `#REF!` 公式错误，全为 NULL
- 伯俊 `stock_qty` 与 `allocatable`（可配）数值基本一致，口径以 `stock_qty` 为准
- 各表列注释（`COMMENT ON COLUMN`）里保留了中文原始表头，可在 psql `\d+` 或 DBeaver 中查看
