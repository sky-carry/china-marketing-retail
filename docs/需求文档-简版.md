# 库存核对平台 · 需求文档（简版）

## 一、背景与痛点

即时零售业务需要定期核对三方库存：伯俊（线下实际库存）、京东、美团（平台在售库存）。此前全靠人工：

1. **纯手工操作**：每次核对需分别从京东、美团后台和伯俊系统手动下载库存数据，再人工比对；
2. **Excel 撑不住**：数据量增长后（平台库存超 20 万行），Excel 公式跨表匹配计算卡顿、频繁出错，实际已不可用；
3. **排查靠逐条翻**：发现差异后无法定位原因，只能一条一条人工核查，极其耗时；
4. **被动响应**：没有主动核对手段，只能等经销商发现库存问题（超卖、缺货）来找我们，问题暴露滞后。

## 二、目标

变被动为主动：定期自动核对三方库存，提前发现差异，并对经销商的库存维护情况形成量化统计。

## 三、平台能力

1. **统一入库**：伯俊、京东、美团库存及飞书门店主数据统一导入数据库，替代手工比对；
2. **自动核对**：门店、商品两级自动匹配，差异自动打标（一致/超卖/缺配/未匹配等），过滤虚拟占位库存；
3. **维护率统计**：按经销商（客户）统计库存一致率，可逐级下钻到具体门店、具体商品，差异原因一目了然；
4. **看板与导出**：交互式看板支持筛选、搜索、排序，一键导出 Excel 明细，局域网内直接访问。

## 四、价值

- 每周核对从人工 1~2 天压缩到分钟级；
- 差异定位从逐条排查变为看板直达门店/单品；
- 从「等经销商上门」变为「主动发现、主动跟进」，并以维护率数据驱动经销商改善。
